📦 COMPILANDO PARA EXE - GUIA COMPLETO
=====================================

O QUE FOI CRIADO:

✅ build-exe.bat
   - Compila o projeto em um único executável
   - Cria pasta "dist" com o arquivo whatsapp-bot.exe
   - Usa pkg para empacotar tudo (Node.js + dependências + código)

✅ prep-distribution.bat  
   - Prepara a pasta para distribuir
   - Cria "bot-distribuicao" com tudo que cliente precisa
   - Inclui banco de dados default (db.sqlite)

✅ package.json atualizado
   - Agora o pkg sabe incluir o banco de dados nos assets


COMO FUNCIONA:

1. SQLite3 é COMPILADO dentro do exe
   → Não precisa instalar nada
   → Funciona em máquina fraca

2. Banco de dados separado (data/db.sqlite)
   → Cada cliente pode ter seu próprio banco
   → Você distribui: exe + banco

3. Estrutura final do cliente:
   
   bot-distribuicao/
   ├── whatsapp-bot.exe       (programa único)
   ├── data/
   │   └── db.sqlite          (banco default que você prepara)
   ├── config.json            (config default)
   ├── public/                (interface web)
   ├── INICIAR.bat            (atalho para rodar)
   └── LEIA-ME.txt            (instruções)


FLUXO DE COMPILAÇÃO E DISTRIBUIÇÃO:

PASSO 1: Prepare o código local
   - Configure config.json como achar melhor
   - Ou deixe em branco para cliente configurar
   - npm install (já deve estar feito)

PASSO 2: Execute build-exe.bat
   - Vai compilar e criar dist/whatsapp-bot.exe
   - Leva alguns minutos na primeira vez
   - Resultado: executable único de ~100MB

PASSO 3: Execute prep-distribution.bat
   - Cria pasta "bot-distribuicao" pronta
   - Contém: exe + dados default + tudo que precisa
   - Pronto para zipar e enviar

PASSO 4: Distribua
   - Zip a pasta "bot-distribuicao"
   - Envie para cliente
   - Cliente extrai e clica em INICIAR.bat

PASSO 5: Cliente usa
   - Clica INICIAR.bat
   - Apareça QR code
   - Escaneia com celular
   - Acessa http://localhost:3000/admin
   - Login: admin/admin


VANTAGENS DA ESTRUTURA:

✅ Usuário leigo: só precisa clicar
✅ Máquina fraca: tudo compilado, sem overhead
✅ Distribuição: uma pasta só
✅ Cada cliente: seu próprio banco de dados
✅ Sem dependências: zero instalação

❌ Desvantagem: exe é grande (~100-150MB)
   → Normal, porque inclui Chromium (necessário para WhatsApp)
   → Compensa porque "just works"


PARA FAZER AGORA:

1. Abra terminal na pasta do projeto
2. Execute: build-exe.bat
3. Aguarde compilação (5-15 minutos)
4. Execute: prep-distribution.bat
5. Pronto! Pasta "bot-distribuicao" está pronta

Se quiser testar antes:
   - Clique em INICIAR.bat dentro de bot-distribuicao
   - Deve funcionar igual ao npm start


CUSTOMIZANDO PARA CLIENTES:

Se cada cliente quer config diferente:

Opção A (Melhor):
   1. Mantenha base de distribuição
   2. Para cada cliente, copie pasta bot-distribuicao
   3. Edite config.json em cada cópia
   4. Crie um novo db.sqlite (pode deixar vazio, cria automaticamente)
   5. Distribua

Opção B (Avançado):
   1. Para cada cliente, customize config.json
   2. Rode localmente para criar db.sqlite com dados de teste
   3. Copie a pasta bot-distribuicao + config + db.sqlite customizados
   4. Distribua


TROUBLESHOOTING:

"build-exe.bat não encontra pkg"
   → Execute no terminal: npm install --global pkg
   → Depois tente novamente

"Compiled exe é grande demais"
   → Normal! Chromium + Node.js = 100-150MB
   → É o tamanho mínimo para funcionar

"db.sqlite não foi copiado"
   → Rode o bot uma vez com npm start
   → Cria automaticamente data/db.sqlite
   → Depois execute prep-distribution.bat novamente

"Cliente roda exe e não aparece terminal"
   → Coloque INICIAR.bat em vez de rodar direto
   → Ou edite para adicionar pause

"Banco de dados vazio no cliente"
   → Normal! Primeira vez não tem dados
   → Começa a registrar quando bot rodar
